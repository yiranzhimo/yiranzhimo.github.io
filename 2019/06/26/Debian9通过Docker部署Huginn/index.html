Debian9通过Docker部署Huginn

Posted on 2019-06-26 Edited on 2020-04-22 In Sever

文章主要介绍的是在通过Docker在Debian9部署Huginn服务。

安装之初，尝试按照官方文档进行安装，
服务器采用的是Debian8，
但是历经好几天，仍然没有解决很多问题，
遂放弃，采用Docker安装。

安装Docker

在安装Docker时，发现了一个问题，Docker不支持Debian8，
因此重新部署服务器，采用Debian9。

sudo:command not found
需要安装sudo
输入命令：
apt-get install sudo -y

安装DockerCE

更新 apt 软件包缓存，并安装 docker-ce。
输入命令：
sudo apt-get update
输入命令：
sudo apt-get install docker-ce

使用脚本安装

在测试或开发环境中 Docker 官方为了简化安装流程，提供了一套便捷的安装脚本，Debian 系统上可以使用这套脚本安装：

启动 Docker CE

输入命令：
sudo systemctl enable docker
输入命令：
sudo systemctl start docker

建立 Docker 用户组

默认情况下，docker 命令会使用 Unix socket 与 Docker 引擎通讯。而只有 root 用户和 docker 组的用户才可以访问 Docker 引擎的 Unix socket。出于安全考虑，一般 Linux 系统上不会直接使用 root 用户。因此，更好地做法是将需要使用 docker 的用户加入 docker 用户组。
建立 docker 组：
sudo groupadd docker
将当前用户加入 docker 组：
sudo usermod -aG docker $USER

Huginn部署

安装过程

Huginn部署

输入命令：
docker search huginn
搜索所有公开的 huginn 镜像
找到huginn/huginn
拉取镜像，输入命令：
docker pull huginn/huginn

运行Huginn

拉取完 huginn 之后，我们其余的环境配置（比如 mysql 安装等等）都不需要再配置了，镜像里面都有了，所以我们要做的就是直接运行 huginn。
输入命令：
docker run -itd --name huginn -p 3000:3000 huginn/huginn
