Mac本地安装KindleEar

Posted on 2019-06-26 Edited on 2020-04-22 In KindleEar Valine:

文章主要介绍的是Mac本地安装KindleEar的过程。

KindleEar 的调试环境

由于 KindleEar 的运行依赖于 Google App Engine 环境，无法像用 Calibre 测试 Recipe 脚本那样直接在本地运行，所以为了方便测试编写

安装 App Engine SDK

[https://storage.googleapis.com/appengine-sdks/featured/GoogleAppEngineLauncher-1.9.85.dmg](macOS 系统)

获取 KindleEar 源代码

将代码Git clone回本地
git clone https://github.com/cdhigh/KindleEar.git

本地运行 KindleEar

Mac需要定位到 KindleEar 项目目录（注意替换 YOURNAME）
cd /Users/YOURNAME/Desktop/KindleEar
输入以下命令让 KindleEar 运行起来（注意有两个空格）
dev_appserver.py ./app.yaml ./module-worker.yaml
看到以下提示，就说明 KindleEar 已经在本机正常运行了

打开浏览器（推荐用 Chrome），输入 http://localhost:8080 即可访问运行在本机上的 KindleEar 程序，输入默认的用户名和密码 admin 即可登入控制界面。至此，KindleEar 的调试环境便准备好了。

常见问题

在这一过程中，Mac系统会有一些问题，KindleEar会默认采用python2.7.x版本，但是Mac自带的python2.7.10版本低于项目需要的python2.7.12版本，因此需要安装其他的python版本。
通过homebrew安装
输入命令：
brew install python2
会安装最高的python2.7.x版本,解决问题。
