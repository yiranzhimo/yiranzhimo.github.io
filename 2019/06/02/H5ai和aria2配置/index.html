H5ai和aria2配置

Posted on 2019-06-02 Edited on 2020-04-21 In Sever Valine:

文章主要介绍的是h5ai的设置
和对aria2的设置。

h5ai配置

h5ai功能设置

这些设置都是在h5ai的默认设置文件中进行修改即可。
默认设置文件路径为：
_h5ai/private/conf/options.json

设置默认语言

在配置文件中搜索l10n找到设置，将en改为zh-cn即可

开启文件搜索功能

在配置文件中搜索search，将false改为true。

多文件下载支援设置

在配置文件中搜索select，将false改为true。

之后h5ai在点击文件的时是选中文件，下载的按钮在左上角。
如果要下载文件呢，可以点击文件名字。

在配置文件中搜索download，将false改为true

设置打包文件的格式为zip，而不是默认的tar的话，需要将php-tar改为shell-zip。

二维码下载

可以通过扫码的方式下载文件。
在配置文件中搜索info，将false改为true，这里要修改两处。

设置登录密码

在BT面板左侧的“文件”中打开
/网站根目录/_h5ai/public
编辑里面的index.php。
1、在头部< ?php的下一行，增加
auth();

2、在底部增加
function auth () { $valid_passwords = array ("账号" => "密码"); $valid_users = array_keys($valid_passwords); $user = $_SERVER['PHP_AUTH_USER']; $pass = $_SERVER['PHP_AUTH_PW']; $validated = (in_array($user, $valid_users)) && ($pass == $valid_passwords[$user]); if (!$validated) { header('WWW-Authenticate: Basic realm="My Realm"'); header('HTTP/1.0 401 Unauthorized'); die ("Not authorized"); } }
注意：
账号和密码可以随便填。

部分文件加密

安装htpasswd工具

输入命令，进行安装：

yum -y install httpd-tools

设置用户名和密码

把用户名、密码保存到指定文件中：(coderschool为用户名，可以自行更改)
输入：
htpasswd -c /usr/local/nginx/htpasswd coderschool
出现：
New password:
在输入密码时，不会有显示的。
需要再次输入：
Re-type new password:
到这就okay了。
Adding password for user coderschool

修改nginx配置文件

修改网站的nginx的配置文件，
在最后加上代码。
location ~ /me { auth_basic "site"; auth_basic_user_file /usr/local/nginx/htpasswd; }
/me为设置的目录名称，
“site”为返回的注释。

重启lnmp即可

aria2

aria2命令

进入下载脚本的目录并运行脚本：./aria2.sh
启动：service aria2 start
停止：service aria2 stop
重启：service aria2 restart
查看状态：service aria2 status
配置文件：/root/.aria2/aria2.conf
下载目录：/usr/local/caddy/www/aria2/Download
（该目录为Github下载安装的，而备用地址下载的默认为/usr/local/caddy/www/file）

添加BTtracker

实现加速下载BT种子。

确认Aria2安装路径

为：/root/aria2
如果不是的话，首先通过find命令找到路径
再通过mv命令将它移动到目录下。
Aria2配置文件路径：/root/.aria2/aria2.conf

关停aria2服务

输入命令：
service aria2 stop

创建Aria2 bt-tracker列表更新脚本

vi /root/trackers-list-aria2.sh
内容如下：

授权脚本

输入命令：
chmod +x /root/trackers-list-aria2.sh

启动aria2

输入命令：service aria2 start

之后打开AriaNG查看BT tracker服务器列表。
