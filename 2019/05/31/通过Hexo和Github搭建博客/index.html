通过Hexo和Github搭建博客

Posted on 2019-05-31 Edited on 2020-04-21 In Blog

文章将从两个方面来阐述，
分别是Hexo和Github。

Hexo

安装

使用 npm 安装 Hexo，
输入：sudo npm install -g hexo-cli
这里有个小坑：Hexo 官网上的安装命令是
npm install -g hexo-cli，
安装时不要忘记前面加上sudo，否则会因为权限问题报错。

创建目录

输入：hexo init blog创建目录
输入：cd blog进入目录
安装npm，输入：npm install
安装完毕以后，初始化查看，
输入：hexo s
浏览器中打开网址(默认是4000端口)
http://localhost:4000
会看到Hexo的默认页面。

Github

得有一个Github账号，之后再操作。

创建仓库

登录您的 GitHub 账号，新建名称为您的用户名.github.io的仓库。
这里强调一点：
假设我的 GitHub 账号的用户名是yiranzhimo，
那么我就应该新建名称为yiranzhimo.github.io的仓库。
Description可写可不写，随意。
勾上Initialize this repository with a README。
点击Create Repository完成创建。

开启GitHub Pages

按照下面这张图片创建

修改全局配置文件

找到本地blog文件夹下_config.yml，在最后加入一下代码。
deploy: type: git repository: https://github.com/xxx/xxx.github.io.git branch: master
记得把repository后的所有xxx换成用户名。

然后，您需要为自己配置身份信息，
终端输入yourname和youremail换成您自己的 GitHub 用户名和邮箱)：
git config --global user.name "yourname" git config --global user.email "youremail"

终端cd到blog文件夹下执行生成静态页面命令：
hexo g

然后进入当前目录下，终端输入：
hexo d
完成上传之后，通过http://xxx.github.io/
(xxx换成您自己的仓库名，也就是用户名)来访问自己刚刚上传的网站。
