Mac 应用已损坏或来自身份不明的开发者的解决方案

Posted on 2020-04-21 Edited on 2020-05-23 In Mac

文章主要介绍的是提示 Mac 应用已损坏或来自身份不明的开发者的解决方案。

在 Mac 电脑中安装非 App Store 软件时，经常会遇到一个问题：应用已损坏或者来自未知的开发者打不开。因为Mac电脑启用了安全机制，默认只信任Mac App Store下载的软件以及拥有开发者 ID 签名的软件。

应用已损坏

macOS Mojave 10.14及以下系统

打开终端，输入命令：
sudo spctl --master-disable

macOS Catalina 10.15系统

打开终端，输入命令：
sudo xattr -rd com.apple.quarantine /Applications/XXX.app

来自未知的开发者

在 Application里面，找到要打开的软件。按住control键，点击应用打开（或者右键打开）即可。

“xxx”将对您的电脑造成伤害。 您应该将它移到废纸篓。

Finder - 应用程序 - xxx右键 - 显示简介 - 覆盖恶意软件保护

Mac 应用程序崩溃

检验签名

打开终端，输入：
sudo -s
输入开机密码，输入：
xattr /Applications/xxx.app
结果显示：
com.apple.quarantine
说明此软件的签名在，正常情况是不会崩溃的，如果崩溃了，那就得清除这个签名。

清除签名

打开终端，输入：
xattr -r -d com.apple.quarantine /Applications/Winclone.app

软件签名

sudo codesign -f -s - --deep /Applications/xxx.app
提示：
/app path : replacing existing signature
即可。
